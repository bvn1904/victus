🎯 FIVE KEY IMPROVEMENTS
════════════════════════════════════════════════════════════════════════════════

1. ✅ SNOOZE FUNCTIONALITY (24 Hours)
   ├─ Uses SecureStore to remember when user clicked "Later"
   ├─ Skips update checks for 24 hours
   └─ Prevents nagging on every app restart

2. ✅ SEPARATED NATIVE vs OTA CHECKS
   ├─ Each has isolated try/catch block

   └─ Graceful fallback if one check fails

   ├─ Step 1: "Download now?" - User decides

4. ✅ BETTER ERROR HANDLING
   ├─ Specific error messages for each failure type
   ├─ Helpful guidance ("try again later")
   └─ Doesn't cascade failures across systems

5. ✅ CORRECT RESTART METHOD
   ├─ Uses Updates.reloadAsync() (Expo standard)
   ├─ Gracefully restarts JS runtime
   ├─ Loads new code bundle from disk
   └─ Cross-platform compatible (iOS + Android)


⚡ WHY OTA REQUIRES RESTART (Your Question)
════════════════════════════════════════════════════════════════════════════════

THE ISSUE:
  When you download an OTA update (fetchUpdateAsync), the NEW code lands on disk,
  but the APP IS STILL RUNNING the OLD code in memory.

THE SOLUTION:
  Updates.reloadAsync() does:
  1. Stop JS execution
  2. Unload old code from RAM
  3. Load new code from disk into RAM
  4. Restart runtime with new code

WITHOUT RESTART:
  ❌ New code on disk
  ❌ Old code in memory (still running)
  ❌ User sees NO changes
  ❌ Update defeated the purpose

WITH RESTART:
  ✅ New code on disk
  ✅ New code in memory
  ✅ User sees changes immediately
  ✅ Update works as intended

ANALOGY:
  Like restarting your computer after a software update - the new version must
  load into memory to take effect.

❓ FAQ
════════════════════════════════════════════════════════════════════════════════

Q: Why separate native and OTA checks?
A: Resilience. If GitHub goes down, Expo OTA still works.

Q: Can we remove the restart requirement?
A: No. Code must be in memory to execute. Restart is necessary.

Q: Does restart lose user data?
A: Only transient state (expected). Persistent data preserved.

Q: What if user ignores "Restart"?
A: Update stays cached. Works on next app restart.

Q: Is this Android-only?
A: No. Works on iOS and Android. Uses Updates.reloadAsync().

Q: Why 24 hours for snooze?
A: Configurable. Change SNOOZE_DURATION constant to different value.
